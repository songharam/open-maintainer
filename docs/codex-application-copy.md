# 신청서용 문구

## 프로젝트명

Open Maintainer Workbench

## 짧은 소개

GitHub 저장소 URL만 입력하면 오픈소스 메인테이너에게 필요한 이슈 분류, 이번 주 우선순위 브리프, PR 리뷰 체크리스트, GitHub 저장소 운영 체크리스트, 릴리스 노트, README 개선안, CONTRIBUTING.md 초안, good first issue 추천, 주간 리포트, 500자 제한 지원 신청 답변, API 크레딧 사용 계획, 후속 업데이트 계획, reviewer packet, 전체 Markdown export를 생성하는 웹앱입니다. `Live GitHub`와 `Sample demo` 모드를 모두 제공해 실제 공개 저장소와 GitHub API 없는 예시 워크로드를 함께 보여줄 수 있습니다. Codex 지원을 받으면 실제 GitHub 데이터와 프로젝트별 유지보수 패턴을 바탕으로 이 작업대를 더 정확하고 실용적인 메인테이너 보조 도구로 발전시킬 수 있습니다.

## 문제

많은 오픈소스 메인테이너는 기능 개발보다 이슈 triage, PR 리뷰 준비, 릴리스 문서 작성, 신규 기여자 안내에 더 많은 반복 시간을 씁니다. 이 작업은 중요하지만 자동화가 부족하면 유지보수 부담으로 쌓이고, 신규 기여자의 진입 장벽도 높아집니다.

## 해결

Open Maintainer Workbench는 메인테이너의 반복 작업을 하나의 단순한 화면으로 모읍니다. 현재 데모는 공개 GitHub API로 Issues, Pull Requests, Releases를 읽고, API 제한이나 네트워크 문제가 있으면 예시 데이터로 fallback합니다. 내부 provider 구조를 통해 나중에 토큰 기반 접근과 Codex/API 분석을 붙일 수 있습니다.

## Codex 지원이 필요한 이유

Codex 지원을 받으면 실제 저장소 데이터를 바탕으로 분류 품질과 문서 초안 품질을 빠르게 개선할 수 있습니다. 특히 여러 저장소의 이슈와 PR 패턴을 비교해 프로젝트별 리뷰 체크리스트, 릴리스 노트, CONTRIBUTING 가이드를 더 정확하게 생성하는 방향으로 발전시킬 수 있습니다.

## 기대 효과

- 메인테이너가 반복 문서 작업에 쓰는 시간을 줄입니다.
- 신규 기여자가 좋은 첫 이슈를 빠르게 찾도록 돕습니다.
- 릴리스와 PR 리뷰 품질을 일정하게 유지합니다.
- 작은 오픈소스 프로젝트도 유지보수 워크플로를 쉽게 갖출 수 있습니다.

## 다음 단계

1. 선택적 GitHub token 입력을 메모리 기반으로 지원
2. 저장소별 라벨 규칙과 릴리스 스타일 학습
3. PR 위험 신호와 누락된 테스트 탐지
4. GitHub comment draft와 저장된 리포트 기록 추가
